## 이 레포의 역할

음식점 사장님(거래처)이 모바일·PC에서 식자재를 발주하는 전용 웹 앱입니다. 네이티브 WebView Shell 안에서 실행되는 웹 레이어로, 상품 검색·장바구니·발주 제출·주문 이력·거래내역·취소·반품까지 거래처 업무 전반을 담당합니다.

Nuxt.js 2 SPA, 35개 이상 페이지, 10개 Vuex 모듈, 5단계 Apollo Link 체인으로 구성되며, 6개 환경(Local·Dev·Lab·QA·Stage·Prod)에 독립적으로 배포됩니다.

## 담당 역할 & 기여

### 1. 인증 흐름 & WebView 라우팅

`auth.ts` 미들웨어가 URL 쿼리 파라미터로 초기 인증을 처리합니다. 네이티브 WebView가 앱 진입 시 인증 파라미터가 포함된 URL을 열면, 미들웨어가 이를 감지해 GraphQL로 로그인하고 인증 토큰을 localStorage에 저장합니다.

WebView 특성상 `history.go(-1)` 같은 뒤로가기 동작이 로그인 URL로 돌아가는 문제가 있어, `history.replaceState`로 히스토리 엔트리를 메인 URL로 교체한 뒤 이동합니다.

### 2. Apollo Link 체인 설계

5개 Link를 조합하여 인증·버전 체크·에러 처리·로깅을 요청 파이프라인으로 분리했습니다.

```
Request  →  authLink  →  versionCheckLink  →  logLink  →  errorLink  →  httpLink  →  서버
Response ←                                                                           ←
```

**authLink** — localStorage에서 인증 토큰을 읽어 모든 요청 헤더에 JWT Bearer 토큰을 주입합니다.

**versionCheckLink** — 서버 응답 헤더의 배포 버전과 현재 클라이언트 버전을 비교합니다. 불일치 시 CDN을 추가로 조회하여 "클라이언트 갱신 필요"와 "백엔드 배포 문제"를 구분합니다.

**errorLink** — GraphQL 에러 유형별로 공통 처리를 중앙화했습니다. 토큰 만료 시 자동 재로그인, 비활성 계정 안내 후 로그아웃 등 에러 코드별 처리를 파이프라인에서 일관되게 처리합니다.

### 3. 유통사별 가격 표기 시스템

거래처마다 부가세 표시 방식과 금액 반올림 단위가 다릅니다. 서버가 내려주는 환경 설정 코드를 프론트엔드 계산 로직에 적용하여, 유통사별로 다른 정책(부가세 포함/별도, 원 단위/10원 단위 반올림 등)을 클라이언트에서 처리합니다.

`Big.js`로 모든 금액 연산을 처리하여 부동소수점 오차를 원천 차단합니다.

### 4. 주문 수량 유효성 검증

상품마다 최소 주문 수량, 최대 일별 주문 수량, 재고 한도를 프론트엔드에서 독립적으로 검증합니다. 서버 왕복 없이 즉각 피드백을 주고, 최종 제출 시 서버에서 한 번 더 검증하는 이중 구조입니다.

### 5. 주문 마감 시간 클라이언트 검증

각 상품의 주문 마감 시간을 클라이언트에서 먼저 검증합니다. 마감된 상품은 장바구니에서 "주문 가능 상품"과 분리하여 별도 섹션으로 표시합니다.

### 6. 서비스 워커 (Workbox)

폰트·이미지 자산을 Workbox 전략으로 캐싱하여 네트워크 불안정 환경의 앱 신뢰성을 높였습니다.

```javascript
// plugins/serviceWorker.ts
if ('serviceWorker' in navigator) {
  window.addEventListener('load', () => {
    navigator.serviceWorker.register('/service-worker.js');
  });
}
```

```javascript
// static/service-worker.js (Workbox 6)
const { CacheFirst, StaleWhileRevalidate } = workbox.strategies;

// Google Fonts 스타일시트 — Stale-While-Revalidate
registerRoute(
  ({ url }) => url.origin === 'https://fonts.googleapis.com',
  new StaleWhileRevalidate({ cacheName: 'google-fonts-stylesheets' }),
);

// 폰트 파일 — Cache-First, 1년 유효
registerRoute(
  ({ url }) => url.origin === 'https://fonts.gstatic.com',
  new CacheFirst({
    cacheName: 'google-fonts-webfonts',
    plugins: [
      new CacheableResponsePlugin({ statuses: [0, 200] }),
      new ExpirationPlugin({ maxAgeSeconds: 60 * 60 * 24 * 365, maxEntries: 30 }),
    ],
  }),
);

// 이미지 — Cache-First, 최대 100개 / 7일 유효
registerRoute(
  ({ request }) => request.destination === 'image',
  new CacheFirst({
    cacheName: 'images',
    plugins: [
      new CacheableResponsePlugin({ statuses: [0, 200] }),
      new ExpirationPlugin({ maxEntries: 100, maxAgeSeconds: 60 * 60 * 24 * 7 }),
    ],
  }),
);
```

## 아키텍처

```
거래처 주문 앱 (Nuxt.js 2 SPA)
│
├── middleware/     → 인증·UA 분기·공지 체크·모니터링 초기화
│
├── apollo/link/    → 인증·버전 체크·로깅·에러 처리 파이프라인
│
├── store/ (Vuex)   → 유통사 환경 설정·주문 상태·거래내역 필터
│
├── utils/calc/     → 금액 계산·수량 검증·날짜 처리
│
└── 네이티브 WebView Shell (별도 레포)
```

## 성과

- Apollo Link 체인으로 인증·버전체크·에러 처리를 파이프라인화 — 페이지 컴포넌트에서 공통 관심사 제거
- versionCheckLink의 CDN 이중 검증으로 "클라이언트 갱신 필요" vs "백엔드 배포 이상"을 자동 구분
- 유통사별 부가세·반올림 전략을 서버 코드 기반으로 처리 — Big.js로 금액 정확성 확보
- Workbox 서비스 워커 도입으로 폰트·이미지 캐싱 — 네트워크 불안정 환경의 앱 신뢰성 개선
