## 이 레포의 역할

유통사(도매상)가 거래처 관리, 발주 수신·처리, 상품 관리, 배송 일정 관리, 세금계산서·정산 업무까지 처리하는 B2B 웹 서비스입니다. 마켓봄 에코시스템에서 가장 많은 기능을 담당하는 메인 프로덕트입니다.

## 담당 역할 & 기여

### 핵심 업무 기능 개발

유통사의 일상 업무 흐름 전반을 지원하는 기능을 구현했습니다.

- **발주 수신 및 처리**: 거래처로부터 들어오는 발주를 실시간으로 수신하고, 확인·처리·취소 워크플로우 구현
- **거래처별 단가 관리**: 거래처마다 다른 상품 단가를 설정하고 일괄 적용하는 가격 관리 인터페이스
- **배송 일정 관리**: 날짜별·지역별 배송 일정 조회 및 도서산간 배송비 그리드 연동
- **출고 지시서 관리**: 출고 지시서 정렬·필터링 및 출고완료 상태 처리

### 자료 일괄 등록 기능 개발

유통사가 상품·단가·거래처 정보를 엑셀 파일로 일괄 등록하는 기능을 담당했습니다.

- **파일 버전 관리**: 서식 버전을 서버에서 관리하고, 구버전 서식 업로드 시 사용자에게 안내 메시지 표시
- **유효성 검증**: 업로드 불가 확장자, 단가 그룹 50개 초과 등 엣지 케이스에 대한 프론트엔드 예외 처리
- **사용자 정보 연동**: 파일 처리 결과에 업로드한 사용자 정보를 포함하여 이력 추적 가능

### 세금계산서 & 거래금액 비교 화면 개발

유통사가 세금계산서와 실제 거래금액을 대조·검증하는 화면을 구현했습니다. 조회 기간 설정, 결제 수단별(가상계좌·신용카드) 금액 분류, 매입처 탭의 지급금액 컬럼 구성, 조회 기준 툴팁 등 세부 UI를 담당했습니다.

### 거래전표 등록 기능 개발

거래전표 등록 화면에서 발생하는 복잡한 상태(삭제된 상품 포함 전표, 인쇄 흐름 등)를 처리했습니다. 전표 인쇄 시 저장 완료 메시지와 인쇄 화면 전환 타이밍 문제를 수정했습니다.

### Apollo Client + GraphQL 데이터 레이어

Apollo Client를 통해 GraphQL API와 통신합니다. 복잡한 쿼리 캐싱 전략을 활용하여 페이지 전환 시 네트워크 요청을 최소화하고, `optimisticResponse`로 UI 응답성을 개선했습니다.

### RealGrid 기반 고성능 테이블

대용량 발주·단가·정산 데이터를 처리하는 화면에서 RealGrid를 활용합니다. 그리드 내 인라인 편집, 다중 선택, 컬럼 고정 등 스프레드시트 수준의 인터랙션을 제공합니다.

## 기술 선택 이유

| 기술 | 선택 이유 |
|------|----------|
| **Nuxt.js + Vue.js** | 팀 내 Vue 기반 기술 스택 표준화, SPA 모드로 B2B 내부 도구에 최적화 |
| **Apollo Client** | GraphQL 캐싱·페이지네이션·optimistic update를 선언적으로 처리 |
| **RealGrid** | 대용량 행 렌더링과 인라인 편집이 필요한 발주·단가 화면에 최적화된 그리드 |
| **TypeScript** | 대규모 코드베이스에서 타입 안전성 확보 — 특히 복잡한 발주·정산 데이터 구조 관리 |
| **XLSX** | 엑셀 일괄 등록 파일 파싱 및 내보내기 |

## 성과

- 발주 수신부터 처리까지의 워크플로우 디지털화로 운영 효율 개선
- 자료 일괄 등록 파일 버전 관리 도입으로 서식 불일치 오류 감소
- 세금계산서·거래금액 비교 화면으로 유통사의 수작업 대조 업무 자동화
- Apollo 캐싱 전략 최적화로 반복 조회 시 네트워크 요청 감소
