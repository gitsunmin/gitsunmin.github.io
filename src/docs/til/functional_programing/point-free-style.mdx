# Point Free Style

Point-free 스타일은 함수의 인자를 명시적으로 사용하지 않는 프로그래밍 스타일입니다. 대신 함수를 다른 함수와 조합하여 작동하게 합니다.

### 예시 1: 명시적 인자 사용

```typescript
const add = (x: number, y: number): number => x + y;
const increment = (x: number): number => add(x, 1);
```

### 예시 2: Point-Free 스타일

```typescript
const add =
  (x: number) =>
  (y: number): number =>
    x + y;
const increment = add(1);
```

위의 예시에서, `increment` 함수는 인자를 명시적으로 사용하지 않고 `add(1)`을 통해 정의됩니다.

---

**장점:**

1. **간결성**: 일단 익숙해지면, point-free 스타일은 코드를 더 간결하게 만들 수 있습니다.
2. **함수의 조합**: 이 스타일은 함수의 조합을 강조하므로 코드의 의도가 더 명확해질 수 있습니다.

**단점:**

1. **가독성**: Point-free 스타일은 때때로 코드의 가독성을 저하시킬 수 있습니다. 너무 복잡한 함수 조합은 다른 개발자들에게 혼란을 줄 수 있습니다.

---

결론적으로, point-free 스타일은 TypeScript에서도 사용할 수 있지만, 언제 어떻게 사용할지를 잘 판단해야 합니다. 항상 코드의 가독성과 유지보수성을 고려하여 적절한 스타일을 선택하는 것이 중요합니다.
